# Session 2026-05-01: public release + simplification

## What shipped

Three commits to `main` of `TieTieWorkSpace/queue-up-for-claude`:

1. **Initial public release** (commit before this session captured) — ported from
   the private `Claude-Job-Queueing` repo, fresh README + `docs/` tree, dropped
   in-flight planning docs and the dev HTTP spike script.

2. **Codex cleanup pass + cli/web dedup** (commit `9dc9f52`) — applied codex
   findings: removed `--check-interval` dead arg, `should_burn` write-only
   field, an unreachable clamp branch, two over-broad `try/except` blocks, six
   stale "HH:55" comments. Extracted three shared helpers (`queue_ops.list_tasks`,
   `injector.render_task_context`, `logger.read_log_lines`) to dedupe between
   CLI and web. Pyflakes pass: removed five unused imports across `cli.py` /
   `web.py` / `runner.py` / `usage_check*.py` and five pointless `f""` strings.
   Burn thresholds (`BURN_USAGE_THRESHOLD_PCT`, `BURN_RESET_WINDOW_MIN`) now
   imported by runner instead of re-hardcoded.

3. **Drop Playwright fallback; HTTP backend only** (commit `8b935ea`) — removed
   the entire browser-scraping path. `usage_check.py` shrank from 651 → 173 LOC.
   Deleted `scripts/check_usage.py` and `tests/test_parse_usage_text.py`.
   Dropped `playwright` from `pyproject.toml`. Verified by uninstalling
   `playwright` from the venv — 132/132 tests still pass. README install
   shrank from 5 steps to 3 (no Chrome, no `playwright install chromium`).

## Key decisions

- **Removed `Playwright` fallback even though it cost some Cloudflare-block
  recoverability.** The dual-backend story complicated docs, install, and
  the dispatcher. Net win for a public repo where install friction matters.
  Documented the tradeoff in `docs/usage-checking.md` under "What this tool
  does NOT do".
- **Kept `Task.status` (write-only field) for disk-grep value.** Codex flagged
  it; we said no.
- **Skipped extracting more shared helpers.** Codex suggested several minor
  dedups that would have been refactors-not-cleanups. Held the line.

## Anti-patterns codex caught and we removed

Add to the review checklist for next session:

- Defensive `try/except KeyError` around code that uses `.get()` only.
- Defensive `try/except IndexError` around code already length-guarded.
- `getattr(result, 'field', None)` for fields always present on the dataclass.
- Function args with default values no production caller ever overrides.
- Unreachable branches in state machines (e.g., `if proposed is not None`
  inside a code path that the producer guarantees `proposed is not None`).
- Stale comments referring to removed features (Playwright, HH:55 schedule,
  `--fork-session`, `os.environ.copy()`).
- `f""` strings without placeholders.
- Re-hardcoded constants when a single source of truth exists.

## Bugs introduced and caught

- One real `TypeError` regression introduced in the cleanup commit: I dropped
  `check_interval_seconds` from `start_runner()` but `web.py:_background_runner`
  still passed it. Codex's second pass caught it before push. Lesson: when
  changing a function signature used by both CLI and web, grep both surfaces.

## Things still on the table (next session candidates)

- The `last_check_backend` field in `RunnerState` and `UsageCheckResult` is
  now always `'http'`. Could drop it on a future major version with a deprecation
  notice; not worth the API-shape break right now.
- `tick_seconds` parameter has only test callers using non-default values.
  Document explicitly in semantic.md; or rename to `_tick_seconds_for_testing`.
- The static `index.html` SPA is 62 KB of un-linted Alpine.js. A pass through
  it would likely find request-cancellation bugs and stale event listeners,
  but no one is asking.
- The `.agent/` system was scaffolded (this session). Future agents should
  use it — `queue-worker compile .` to generate a CLAUDE.md for interactive
  Claude Code use.
